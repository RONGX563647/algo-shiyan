## 相等元素问题 - 哈希表原理

### 问题本质分析

相等元素问题的核心是**元素查找与判重**。我们需要高效地判断一个元素是否已经出现过。

```mermaid
flowchart TD
    A[输入数组] --> B{选择算法}
    B -->|暴力法| C[两层循环比较]
    B -->|排序法| D[排序后比较相邻]
    B -->|哈希法| E[哈希表查找]
    
    C --> F[时间 O n²]
    D --> G[时间 O n log n]
    E --> H[时间 O n 平均]
    
    F --> I[效率最低]
    G --> J[效率中等]
    H --> K[效率最高]
```

### 哈希表核心原理

哈希表通过**哈希函数**将元素映射到固定位置，实现O(1)的平均查找时间。

```mermaid
flowchart LR
    subgraph 哈希函数
        A[元素 value] --> B["hash = abs value % capacity"]
        B --> C[桶索引 index]
    end
    
    subgraph 哈希表结构
        D[桶0] --> E[链表: 5 → 21 → null]
        F[桶1] --> G[链表: null]
        H[桶2] --> I[链表: 16 → 2 → null]
        J[...] --> K[...]
    end
    
    C --> D
    C --> F
    C --> H
    C --> J
```

### 链地址法解决冲突

当多个元素映射到同一桶时，使用链表存储所有冲突元素：

```mermaid
flowchart TD
    subgraph 冲突场景
        A["元素 5: hash = 5 % 10 = 5"]
        B["元素 15: hash = 15 % 10 = 5"]
        C["元素 25: hash = 25 % 10 = 5"]
    end
    
    subgraph 桶5的链表
        D[桶5] --> E[节点5] --> F[节点15] --> G[节点25] --> H[null]
    end
    
    A --> D
    B --> E
    C --> F
```

### 添加元素流程

```mermaid
flowchart TD
    A[添加元素 value] --> B[计算哈希值]
    B --> C[获取桶索引 index]
    C --> D[遍历桶内链表]
    D --> E{元素是否存在?}
    E -->|存在| F[返回 false - 重复]
    E -->|不存在| G[创建新节点]
    G --> H[插入链表头部]
    H --> I[返回 true - 成功添加]
```

### 时间复杂度分析

```mermaid
pie title 哈希表操作时间复杂度分布
    "理想情况 O 1" : 60
    "平均情况 O 1" : 30
    "最坏情况 O n" : 10
```

| 场景 | 时间复杂度 | 说明 |
|------|------------|------|
| 理想情况 | O(1) | 无冲突，直接定位 |
| 平均情况 | O(1) | 少量冲突，链表长度有限 |
| 最坏情况 | O(n) | 所有元素冲突到同一桶 |

### 代码实现关键

```java
public boolean add(int value) {
    int index = hash(value);
    
    Node current = table[index];
    while (current != null) {
        if (current.value == value) {
            return false;
        }
        current = current.next;
    }
    
    Node newNode = new Node(value);
    newNode.next = table[index];
    table[index] = newNode;
    return true;
}
```

## 整数集合分解 - 快速排序原理

### 问题本质分析

整数集合分解的核心是**排序后分组**。要使两个子集和差最大，贪心策略是让一组包含最小元素，另一组包含最大元素。

```mermaid
flowchart TD
    A[原始数组] --> B[排序]
    B --> C[分割]
    C --> D[前半部分 - 较小元素]
    C --> E[后半部分 - 较大元素]
    D --> F[子集S1]
    E --> G[子集S2]
    F --> H[和最小]
    G --> I[和最大]
    H --> J[差值最大]
    I --> J
```

### 贪心策略正确性证明

```mermaid
flowchart TD
    A[贪心策略] --> B[最小元素放S1]
    A --> C[最大元素放S2]
    
    B --> D["设排序后: a0 ≤ a1 ≤ ... ≤ an-1"]
    D --> E["S1 = {a0, ..., an/2-1}"]
    D --> F["S2 = {an/2, ..., an-1}"]
    
    E --> G["证明: 任意交换都会减小差值"]
    G --> H["设交换S2中x与S1中y y<x"]
    H --> I["新差值 = 原差值 - 2 x-y"]
    I --> J["差值减小 故贪心最优"]
```

### 快速排序算法流程

```mermaid
flowchart TD
    A[快速排序] --> B[选择基准 pivot]
    B --> C[分区操作]
    C --> D[小于pivot放左边]
    C --> E[大于pivot放右边]
    D --> F[递归排序左半部分]
    E --> G[递归排序右半部分]
    F --> H[完成]
    G --> H
```

### 分区操作详解

```mermaid
flowchart LR
    subgraph 分区前
        A1[68] --> A2[25] --> A3[34] --> A4[16] --> A5[2] --> A6[37] --> A7[3] --> A8[95 pivot]
    end
    
    subgraph 分区过程
        B[i = -1] --> C[j 遍历 0 to 6]
        C --> D{arr j <= pivot?}
        D -->|是| E[i++ 交换 arr i 和 arr j]
        D -->|否| F[j++]
    end
    
    subgraph 分区后
        G1[2] --> G2[3] --> G3[16] --> G4[25] --> G5[34] --> G6[37] --> G7[68] --> G8[95]
    end
```

### 分区操作步骤分解

```mermaid
sequenceDiagram
    participant Arr as 数组
    participant i as i指针
    participant j as j指针
    participant Pivot as pivot=95
    
    Note over Arr: 初始: [68,25,34,16,2,37,3,95]
    Note over i,j: i=-1, j=0
    
    j->>Arr: arr[0]=68 <= 95?
    Note over i: i++, swap(arr[-1+1], arr[0])
    Note over Arr: i=0, 无变化
    
    j->>Arr: arr[1]=25 <= 95?
    Note over i: i++, swap(arr[1], arr[1])
    Note over Arr: i=1
    
    j->>Arr: 继续遍历...
    Note over Arr: 所有元素 <= 95
    
    Note over i: 最后swap(arr[i+1], pivot)
    Note over Arr: [2,3,16,25,34,37,68,95]
```

1. **选择合适的数据结构** - 哈希表适合查找判重

3. **分析复杂度** - 评估时间和空间开销
4. **自实现的意义** - 深入理解底层原理
