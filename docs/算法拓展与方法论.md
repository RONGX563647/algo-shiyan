## 哈希表进阶：开放地址法

除了链地址法，另一种解决哈希冲突的方法是**开放地址法**。

### 开放地址法类型

```mermaid
flowchart TD
    A[开放地址法] --> B[线性探测]
    A --> C[二次探测]
    A --> D[双重哈希]
    
    B --> E[冲突时找下一个位置]
    C --> F[冲突时找index加1的平方]
    D --> G[使用第二个哈希函数计算步长]
```

### 线性探测示例

```mermaid
flowchart LR
    subgraph hash1 [哈希表容量为7]
        A1["索引0: 空"]
        A2["索引1: 空"]
        A3["索引2: 空"]
        A4["索引3: 空"]
        A5["索引4: 空"]
        A6["索引5: 5"]
        A7["索引6: 空"]
    end
    
    subgraph hash2 [插入12后]
        B1["索引0: 空"]
        B2["索引1: 空"]
        B3["索引2: 空"]
        B4["索引3: 空"]
        B5["索引4: 空"]
        B6["索引5: 5"]
        B7["索引6: 12"]
    end
    
    hash1 --> hash2
```

### 二次探测示例

```mermaid
flowchart TD
    A[插入元素发生哈希冲突] --> B[尝试位置index加1的平方]
    B --> C{仍冲突?}
    C -->|是| D[尝试位置index加2的平方]
    D --> E{仍冲突?}
    E -->|是| F[尝试位置index加3的平方]
    E -->|否| G[成功插入]
    C -->|否| G
```

### 双重哈希

```mermaid
flowchart TD
    A[双重哈希] --> B[hash1等于key对capacity取模]
    A --> C[hash2等于1加key对capacity减1取模]
    A --> D[探测位置等于hash1加i乘hash2]
    
    B --> E[第一个哈希函数确定初始位置]
    C --> F[第二个哈希函数确定步长]
    D --> G[不同key有不同的探测序列]
```

### 链地址法 vs 开放地址法

```mermaid
flowchart TD
    subgraph chain [链地址法]
        A1[优点] --> B1[删除简单]
        A1 --> C1[无聚集问题]
        A1 --> D1[链表动态扩展]
        A2[缺点] --> E1[需要额外指针空间]
        A2 --> F1[链表遍历开销]
    end
    
    subgraph open [开放地址法]
        G1[优点] --> H1[无需额外空间]

        G2 --> K1[聚集问题]
        G2 --> L1[表不能过满]
    end
```

### 聚集问题


## 排序算法对比与优化

### 主要排序算法对比

```mermaid
flowchart TD
    A[排序算法] --> B[冒泡排序]
    A --> C[选择排序]
    A --> D[插入排序]
    A --> E[快速排序]
    A --> F[归并排序]
    A --> G[堆排序]
    
    B --> H[稳定且简单 时间n平方]
    C --> I[不稳定且简单 时间n平方]
    D --> J[稳定 对近乎有序数据高效]
    E --> K[不稳定 平均最快 时间nlogn]
    F --> L[稳定 需额外空间 时间nlogn]
    G --> M[不稳定 原地排序 时间nlogn]
```

### 排序算法复杂度对比表

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|----------|----------|------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |

## 搜索算法：DFS vs BFS

### DFS与BFS对比

```mermaid
flowchart TD
    subgraph dfs [DFS深度优先]
        A1[策略] --> B1[深入一条路径]
        B1 --> C1[使用栈]
        C1 --> D1[找到任意解]
        D1 --> E1[可能不是最短]
    end
    
    subgraph bfs [BFS广度优先]
        A2[策略] --> B2[逐层扩展]
        B2 --> C2[使用队列]
        C2 --> D2[找到最短解]
        D2 --> E2[空间开销大]
    end
```

### 搜索树可视化

```mermaid
graph TD
    A[根节点] --> B[第1层节点1]
    A --> C[第1层节点2]
    A --> D[第1层节点3]
    
    B --> E[第2层节点4]
    B --> F[第2层节点5]
    C --> G[第2层节点6]
    D --> H[第2层节点7]
    
    E --> I[第3层节点8]
    E --> J[第3层节点9目标]
    
    style J fill:#9f9,stroke:#333
```

### DFS搜索路径

```mermaid
sequenceDiagram
    participant Stack as 栈
    participant Node as 当前节点
    participant Found as 是否目标
    
    Note over Stack: 初始包含根节点
    Stack->>Node: pop取出根节点
    Node->>Found: 不是目标
    Found-->>Stack: push添加子节点
    
    Stack->>Node: pop取出子节点
    Node->>Found: 不是目标
    Found-->>Stack: push添加子节点
    
    Note over Stack: 继续深入
```

### BFS搜索路径

```mermaid
sequenceDiagram
    participant Queue as 队列
    participant Node as 当前节点
    participant Found as 是否目标
    
    Note over Queue: 初始包含根节点
    Queue->>Node: dequeue取出根节点
    Node->>Found: 不是目标
    Found-->>Queue: enqueue添加子节点
    
    Queue->>Node: dequeue取出子节点
    Node->>Found: 不是目标
    Found-->>Queue: enqueue添加子节点
    
    Note over Queue: 逐层扩展
```

### 适用场景分析

## 算法设计方法论

### 问题解决流程

```mermaid
flowchart TD
    A[理解问题] --> B[分析输入输出]
    B --> C[识别问题类型]
    C --> D[选择算法策略]
    D --> E[设计算法]
    E --> F[分析复杂度]
    F --> G[实现与测试]
    G --> H{是否满足要求?}
    H -->|是| I[完成]
    H -->|否| J[优化或换策略]
    J --> D
```

### 问题类型识别

```mermaid
flowchart TD
    A[问题类型] --> B[查找问题]
    A --> C[排序问题]
    A --> D[优化问题]
    A --> E[搜索问题]
    A --> F[组合问题]
    
    B --> G[哈希表或二分查找]
    C --> H[快速排序或归并排序]
    D --> I[贪心或动态规划]
    E --> J[DFS或BFS]
    F --> K[回溯或分治]
```

### 算法策略选择

```mermaid
flowchart TD
    A[问题特征] --> B{最优子结构?}
    B -->|是| C{子问题重叠?}
    C -->|是| D[动态规划]
    C -->|否| E[贪心算法]
    
    B -->|否| F{可分解?}
    F -->|是| G[分治算法]
    F -->|否| H[暴力搜索]
    
    H --> I{搜索空间大?}
```

### 贪心算法适用条件

## 总结

### 核心知识点汇总

| 类别 | 知识点 | 关键理解 |
|------|--------|----------|
| 哈希表 | 链地址法/开放地址法 | 冲突处理策略 |
| 排序 | 快速排序优化 | 基准选择、三路分区 |
| 递归 | 主定理 | 递归复杂度分析 |
| 分治 | 分解-解决-合并 | 问题分解策略 |
| 搜索 | DFS vs BFS | 最短路径 vs 任意解 |

### 算法设计流程总结

```mermaid
flowchart TD
    A[问题] --> B[建模]
    B --> C[选择策略]
    C --> D[设计算法]
    D --> E[分析复杂度]
    E --> F[实现]
    F --> G[测试验证]
    G --> H{满足要求?}
    H -->|是| I[完成]
    H -->|否| J[优化]
    J --> C
```
